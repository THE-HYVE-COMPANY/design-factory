# Support

Thanks for using Design Factory. Here is where to go depending on what you need.

## Before you ask

- **Getting started / running locally** — [`docs/quickstart.md`](docs/quickstart.md)
  and the [README](README.md) ([English](README.en.md)).
- **A provider won't connect** — [`docs/providers.md`](docs/providers.md) lists
  every adapter, what it needs (CLI install or API key), and its readiness state.
- **Something is broken at runtime** — [`docs/troubleshooting.md`](docs/troubleshooting.md)
  covers the common failures (bridge not reachable, port conflicts, blank iframe).

## Questions and help

For "how do I…" or "is this supposed to…" questions, open a
[GitHub Discussion](https://github.com/THE-HYVE-COMPANY/design-factory/discussions)
if Discussions are enabled, otherwise open an issue with the **question** label.
Search existing issues and discussions first — your question may already be answered.

## Reporting a bug

Open a [bug report](https://github.com/THE-HYVE-COMPANY/design-factory/issues/new/choose)
using the template. Include your OS, Node version, the provider you used, and the
steps to reproduce. The more specific the repro, the faster the fix.

## Requesting a feature

Open a feature request with the template. Note that scope and direction are
maintainer-led — see [`GOVERNANCE.md`](GOVERNANCE.md) and [`ROADMAP.md`](ROADMAP.md)
for what is in scope and how to influence it.

## Security issues

**Do not** open a public issue for a vulnerability. Follow the private process in
[`SECURITY.md`](SECURITY.md).

## Contributing

Setup, conventions, and the quality gates are in [`CONTRIBUTING.md`](CONTRIBUTING.md).
