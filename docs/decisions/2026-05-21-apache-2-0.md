# Decision · License switch to Apache-2.0

**Date:** 2026-05-21
**Status:** Accepted — supersedes the 2026-05-15 MIT decision recorded in `docs/decisions/design-factory-v1-beta-decisions.md`.
**Affects:** `LICENSE`, `NOTICE`, `package.json`, README footer (EN + PT).

## Context

The repo started under Apache-2.0 in early DF prototyping. On
2026-05-15 the founder approved a swap to MIT for the v1 beta cut,
preferring "minimum permission, maximum permissiveness". The
checklist for the public cut now invites a re-decision because:

1. The public cut will carry the project further than the original
   beta scope — first external contributors, eventual fork
   ecosystems, possible commercial derivatives.
2. AI-adjacent code increasingly attracts patent claims; the MIT
   licence has no explicit patent grant.
3. Brand protection has become a real concern. MIT covers code only;
   any trademark guard has to live outside the licence.
4. The HYVE brand is now meaningful in its own right (the operational
   taste thesis, the canonical design systems, the wordmark in the
   product surface). The licence + NOTICE pair scopes the grant to
   code while leaving the marks reserved.

## Decision

Adopt **Apache License 2.0** as the canonical project licence.

- `LICENSE` carries the full Apache-2.0 text with the appendix
  attached.
- `NOTICE` is added at the repo root with: copyright statement,
  pointer to LICENSE, trademark boundary (HYVE + Design Factory
  marks reserved), and the third-party software statement (no
  copyleft in the resolved dep tree as of this date).
- `package.json` `"license"` set to `"Apache-2.0"`.
- README badges and footer (EN + PT) point at the new LICENSE +
  NOTICE.

## Consequences

**Positive:**

- Explicit, unambiguous patent grant covering every contributor's
  IP — critical for AI-adjacent code that could otherwise be a
  patent magnet.
- Trademark exclusion explicit in Section 6 of the licence, layered
  with an explicit NOTICE clause that names the reserved marks.
- Recognised by GitHub, OSI, and downstream packagers (apt, brew,
  npm, etc.) as a standard permissive licence — no friction for
  forks or commercial use.
- Compatible with the existing dep tree (MIT / Apache-2.0 / ISC /
  BSD-3 dominance — no copyleft conflict).

**Negative:**

- Longer licence text than MIT — every fork has to redistribute the
  full Apache 2.0 file. (Already standard practice for any
  Apache-licensed dep.)
- Section 4(b) requires modified files to carry change notices.
  Practically, forks set this up once via their commit history;
  not a blocker.

**Neutral:**

- The licence change is a one-time event. After this PR, "what
  licence do we ship under" stops being a recurring decision.

## How this supersedes the 2026-05-15 MIT decision

The earlier decision is preserved in
`docs/decisions/design-factory-v1-beta-decisions.md` for the
record. From this date forward, the LICENSE file is the
authoritative source of truth; the MIT decision document is
historical and should not be cited as current canon.

If a contributor reads the older doc and the new LICENSE together,
the LICENSE wins.
