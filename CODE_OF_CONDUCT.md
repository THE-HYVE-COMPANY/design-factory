# Code of Conduct

Design Factory is a small, pragmatic project. We want contributors to feel
welcome and to get useful work done. This document is intentionally short.

## Expected behavior

- Be direct and respectful. Disagreement is fine; personal attacks are not.
- Critique ideas, not people. "This approach breaks on Windows" is good.
  "You don't know what you're doing" is not.
- Assume good intent. If a review comment feels sharper than intended, ask
  for clarification before escalating.
- Keep issues and PRs on-topic. Side conversations belong in Discussions.

## Unacceptable behavior

- Harassment, insults or discriminatory language — including in code
  comments, commit messages and issue threads.
- Sharing someone else's private information without explicit permission.
- Sustained disruption of issues, PRs or discussions.

## Scope

This applies anywhere the project is active: GitHub (issues, PRs,
Discussions, code review), project-affiliated chat channels, and any public
representation of the project.

## Reporting

Use GitHub's private reporting tools where available, or open a GitHub
issue if the report can be handled publicly. Include a summary of what
happened, links to relevant threads, and anything else that would help.
Reports are handled by maintainers with as much privacy as the channel
allows.

## Enforcement

Depending on severity, maintainers may:

1. Privately warn the person involved.
2. Publicly flag the behavior as out of bounds and ask them to stop.
3. Temporarily restrict their ability to interact on the repo.
4. Ban them from the project.

We aim to be proportionate. First-time minor issues usually get a private
conversation, not a public shaming.
